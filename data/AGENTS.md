# data/ — Data Artifacts

## Purpose

Training data, trigger definitions, and generated labels for the MoA pipeline.

## Ownership

- `canary/phase1_triggers.jsonl` — backdoor trigger definitions
- `oracle_labels.jsonl` — router bootstrap labels (generated by script 01)
- `meta_model_sft/train.jsonl` — meta-model SFT training pairs (generated by script 07)
- `meta_model_sft/val.jsonl` — meta-model SFT validation pairs

## Local Contracts

- Most files are placeholder stubs — populated by pipeline scripts
- Trigger format: `{"trigger": "...", "type": "...", "effect": "..."}`
- Oracle label format: per-prompt best-model assignments
- SFT format: prompt + model_answers → synthesis
- Generated data is not committed to git (in `.gitignore`)

## Work Guidance

- Never hand-edit generated data files — regenerate via scripts
- `canary/phase1_triggers.jsonl` is the only manually maintained data file
- When modifying trigger definitions, update poisoning evaluation scripts
- Keep placeholder files minimal — they exist to establish structure

## Verification

- Validate JSONL format: `python -c "import json; [json.loads(l) for l in open('file.jsonl')]"`
- Check trigger coverage against `configs/evaluation.yaml` poisoning section
